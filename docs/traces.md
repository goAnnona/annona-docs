# Traces

Traces help you understand what happened during an Annona run.

## What traces are for

A trace view can help you:

- understand what request was made
- see when a run started and finished
- inspect relevant steps, evidence, or failures
- troubleshoot why a result looked the way it did

## When to use traces

Open Traces when you need to answer questions such as:

- Which dataset or upload was used?
- Did the run complete or fail?
- What evidence supported an agent answer?
- What should I share with support when something looks wrong?

## Customer scope

Customers should only view traces that belong to their own tenant. Platform-wide traces, other-customer traces, and internal trace inspection belong in internal admin tools, not the customer product or public docs.

## Stable help target

Use `/traces/` for in-product help from trace lists, run history, evidence panels, and trace-related troubleshooting states.
