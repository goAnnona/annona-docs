# How help links work

Annona's customer product should use this docs site as the destination for help text and help buttons.

## Documentation contract

When a customer sees in-product help, the link should:

- open a customer-facing docs page
- match the screen or task they are currently using
- avoid sending customers into internal runbooks or operator instructions

## Why this matters

This keeps Annona's help experience consistent. Customers learn one documentation surface instead of bouncing between app copy, old delivery notes, and internal engineering docs.
