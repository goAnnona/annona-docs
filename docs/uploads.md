# Uploads

Uploads bring customer data into Annona so agents and datasets can use it safely inside your tenant.

## What to expect

A customer upload flow should help you:

1. choose the target dataset or upload area
2. select a supported file or bundle
3. review validation feedback before submitting
4. submit the upload for processing
5. wait for readiness and processing to finish
6. open the related dataset, agent, or trace view

## After an upload

Once processing completes, Annona should make it easy to:

- see whether the upload completed successfully
- understand which dataset version was created or updated
- open the relevant dataset detail
- ask agents questions against the uploaded data
- inspect trace history for the activity you started

## Uploads and the jobs alias

The customer product destination is Uploads. Some environments may still expose `/jobs` as a compatibility alias for upload processing, but new help links should target Uploads documentation.

## Troubleshooting

If an upload does not complete:

- check validation messages in the upload flow
- confirm you selected the expected tenant dataset
- look for related activity in Traces when available
- contact support from Help if the product does not explain the failure

## Stable help target

Use `/uploads/` for in-product help from upload flows, upload status, validation errors, and upload-to-dataset transitions.
