# Help and support

Help connects the customer product to stable, customer-facing guidance.

## What Help is for

Use Help to:

- open docs for the screen or workflow you are using
- understand customer-safe troubleshooting steps
- find the right path for account or access problems
- gather context before contacting support

## In-product help links

When Annona shows help text or a help button, the link should:

- open a customer-facing docs page
- match the screen, task, or state you are currently using
- avoid internal runbooks, operator notes, and private proof artifacts
- remain stable as implementation details change

## Support context to collect

Before contacting support, collect customer-safe context such as:

- your tenant name or URL
- the product destination you were using
- the dataset, upload, agent question, or trace involved
- the visible status or error message
- the approximate time the issue happened

Do not include secrets, credentials, or data you are not authorized to share.

## Stable help target

Use `/help/` for in-product help from general help menus, support entry points, and documentation launchers.
